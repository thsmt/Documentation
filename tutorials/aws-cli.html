AWS CLI 実務手順集

AWS CLIを使った認証確認、プロファイル切替、CloudFormation操作、ログ確認の基本手順を整理します。

補足 AWS CLIの作業前には、利用者、アカウントID、リージョンを確認します。複数アカウントを扱う場合は、プロファイル名を明示して操作対象を取り違えないようにします。

インストール

作業端末のOSに合わせてAWS CLIをインストールします。

powershell
winget install --id Amazon.AWSCLI -e --source winget
aws --version
zsh
brew install awscli
aws --version
bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
aws --version

認証情報の確認

powershell
aws sts get-caller-identity

プロファイルの使い分け

powershell
aws configure --profile dev
aws sts get-caller-identity --profile dev

リージョン確認

作業前に対象リージョンを明示します。東京リージョンで作業する場合は ap-northeast-1 を指定します。

powershell
aws configure set region ap-northeast-1 --profile dev

CloudWatch Logs確認

powershell
aws logs describe-log-groups --profile dev
aws logs tail /aws/lambda/example-function --follow --profile dev
