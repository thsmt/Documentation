AWS CLI 実務手順集

AWS CLIを使った認証確認、プロファイル切替、IAM、Security Group、SNS、Elastic IP、Compute Optimizerの基本手順を整理します。

補足 AWS CLIの作業前には、利用者、アカウントID、リージョンを確認します。複数アカウントを扱う場合は、プロファイル名を明示して操作対象を取り違えないようにします。

インストール

作業端末のOSに合わせてAWS CLIをインストールします。

powershell
winget install --id Amazon.AWSCLI -e --source winget
aws --version
zsh
brew install awscli
aws --version
bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
aws --version

認証情報の確認

現在の認証情報が、想定しているAWSアカウントとIAMユーザー/ロールを向いているか確認します。

powershell
aws sts get-caller-identity

プロファイルの使い分け

作業用のプロファイルを作成します。複数アカウントを扱う場合は、以降のコマンドで --profile <PROFILE> を明示します。

powershell
aws configure --profile <PROFILE>

作成したプロファイルで認証先を確認します。

powershell
aws sts get-caller-identity --profile <PROFILE>

リージョン確認

作業前に対象リージョンを明示します。東京リージョンで作業する場合は ap-northeast-1 を指定します。

powershell
aws configure set region ap-northeast-1 --profile <PROFILE>

IAMユーザー・グループ操作

原則として人のログイン用途はIAM Identity Centerを優先します。IAMユーザーを作成する場合は、用途、権限、利用期限、アクセスキーの有無を事前に確認します。

値説明
<USER_NAME>作成または確認するIAMユーザー名
<GROUP_NAME>ユーザーを所属させるIAMグループ名
<PROFILE>AWS CLIの作業用プロファイル名

IAMユーザーを作成します。

powershell
aws iam create-user --user-name <USER_NAME> --profile <PROFILE>

作成したIAMユーザーを既存グループに追加します。

powershell
aws iam add-user-to-group --user-name <USER_NAME> --group-name <GROUP_NAME> --profile <PROFILE>

ユーザーの所属グループを確認します。

powershell
aws iam list-groups-for-user --user-name <USER_NAME> --profile <PROFILE>
確認観点 作業前後でユーザー数、所属グループ、付与ポリシーを比較します。アクセスキーを作成する場合は、保管方法、ローテーション、削除予定日を作業記録に残します。

Security Group ルール追加・削除

Security Group変更は影響範囲が広いため、対象SG、方向、プロトコル、ポート、送信元/宛先、説明欄を事前に確定します。

AWS CLIのSecurity Groupルールで全プロトコルを指定する場合は、公式リファレンスに合わせて --protocol -1 を使用します。

値説明
<SECURITY_GROUP_ID>変更対象のSecurity Group ID
<CIDR>許可または削除する送信元CIDR。例: 203.0.113.10/32
<PROFILE>AWS CLIの作業用プロファイル名

作業前に、対象Security Groupの現在のルールを確認します。

powershell
aws ec2 describe-security-groups --group-ids <SECURITY_GROUP_ID> --profile <PROFILE>

HTTPの80/tcpだけを許可する例です。

powershell
aws ec2 authorize-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol tcp --port 80 --cidr <CIDR> --profile <PROFILE>

全プロトコル・全ポートを許可する例です。検証や一時作業以外では原則として避け、作業後に必ず削除します。

powershell
aws ec2 authorize-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol -1 --cidr <CIDR> --profile <PROFILE>

80/tcpの許可ルールを削除する例です。

powershell
aws ec2 revoke-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol tcp --port 80 --cidr <CIDR> --profile <PROFILE>

全プロトコル・全ポートの許可ルールを削除する例です。

powershell
aws ec2 revoke-security-group-ingress --group-id <SECURITY_GROUP_ID> --protocol -1 --cidr <CIDR> --profile <PROFILE>

SNS Topic・Subscription作成

通知先を作成する場合は、Topic、Subscription、確認メールの承認、通知テストまでを一連の手順として扱います。

SNS Topicを作成します。

powershell
aws sns create-topic --name <TOPIC_NAME> --profile <PROFILE>

メール通知先をSubscriptionとして登録します。

powershell
aws sns subscribe --topic-arn <TOPIC_ARN> --protocol email --notification-endpoint <MAIL_ADDRESS> --profile <PROFILE>

通知テストを実行します。

powershell
aws sns publish --topic-arn <TOPIC_ARN> --message "notification test" --profile <PROFILE>

Elastic IP 解放後の再取得

Elastic IPを誤って解放した場合でも、他のAWSアカウントに割り当てられていなければ、AWS CLIで同じPublic IPv4アドレスを指定して再取得できる場合があります。詳細は Amazon EC2 User Guide を確認します。

値説明
<PUBLIC_IP>再取得したいElastic IPアドレス
<REGION>Elastic IPを利用していたリージョン
<PROFILE>AWS CLIの作業用プロファイル名

解放済みのElastic IPを同じアドレスで再取得します。

powershell
aws ec2 allocate-address --domain vpc --address <PUBLIC_IP> --region <REGION> --profile <PROFILE>

再取得したElastic IPをEC2インスタンスへ関連付ける場合は、払い出されたAllocation IDを指定します。

powershell
aws ec2 associate-address --allocation-id <ALLOCATION_ID> --instance-id <INSTANCE_ID> --region <REGION> --profile <PROFILE>
注意 解放済みのElastic IPが他のAWSアカウントに割り当てられている場合は再取得できません。作業前に対象リージョン、対象IP、現在の割り当て状況を確認します。

Compute Optimizer オプトアウト

Compute Optimizerからアカウントをオプトアウトする場合は、AWS CLIでEnrollment statusをInactiveに更新します。詳細は AWS Compute Optimizer User Guide を確認します。

値説明
<PROFILE>AWS CLIの作業用プロファイル名

現在のCompute Optimizer登録状態を確認します。

powershell
aws compute-optimizer get-enrollment-status --profile <PROFILE>

Compute Optimizerからアカウントをオプトアウトします。

powershell
aws compute-optimizer update-enrollment-status --status Inactive --profile <PROFILE>
注意 オプトアウト時に --include-member-accounts を指定するとエラーになります。また、オプトアウトすると対象アカウントのレコメンデーションと関連メトリクスデータはCompute Optimizerから削除されます。
