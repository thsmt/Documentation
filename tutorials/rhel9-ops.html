RHEL9 手動構築手順

AWS EC2上のRHEL9を手動構築する際の初期パッケージ導入とlogrotate設定を整理します。実環境名、IPアドレス、顧客固有値は扱わず、手順だけを再利用できる形にまとめます。

前提 PDFの手動構築手順をもとに、公開できる範囲で一般化しています。root権限で実行する前提のため、コマンド例では sudo を省略しています。コピーしやすさを優先し、プロンプト記号の # はコードブロックに含めません。

作業前確認

観点確認内容
対象確認アカウント、リージョン、インスタンスID、OSバージョン、ホスト名を確認します。
退避AMI、EBS Snapshot、設定ファイルのバックアップ要否を確認します。
接続経路Session Manager、踏み台、Security Group一時許可のどれを使うか確認します。
影響サービス再起動、パッケージ更新、カーネル更新の有無を確認します。

基本パッケージのインストール

タブ補完、疎通確認、ファイル取得、ネットワーク調査、端末操作で利用する基本パッケージをインストールします。

root shell
yum install -y telnet lftp mlocate libyaml wget mtr tcpdump tmux bash-completion net-tools
補足 bash-completion 導入後、タブ補完がオプションまで補完されない場合があります。その場合は再ログイン、または必要に応じてOS再起動後に確認します。

logrotate設定

既存のlogrotate設定を退避します。

root shell
cp -pi /etc/logrotate.conf /etc/logrotate.conf.org

logrotate設定を編集します。

root shell
vi /etc/logrotate.conf

/etc/logrotate.conf を以下の内容で上書きします。

conf
# see "man logrotate" for details
# rotate log files weekly
weekly

# keep 4 weeks worth of backlogs
rotate 54

# create new (empty) log files after rotating old ones
create

# use date as a suffix of the rotated file
dateext

# uncomment this if you want your log files compressed
compress

# RPM packages drop log rotation information into this directory
include /etc/logrotate.d

# system-specific logs may be also be configured here.
確認観点 上記のコメントは標準設定由来の文言を含みます。rotate 54 は保持世代数を54にする設定のため、実環境でコメントと設定値の整合性を重視する場合はコメントもあわせて修正します。

ミドルウェア別logrotate設定

ミドルウェアに合わせて、/etc/logrotate.d/ 配下の設定ファイルを適宜変更します。

root shell
ls -l /etc/logrotate.d/

対象ミドルウェアのlogrotate設定を編集します。

root shell
vi /etc/logrotate.d/<SERVICE_NAME>
注意 /etc/logrotate.d/ 配下にバックアップファイルを置くと、バックアップファイル内の記述もlogrotateの対象として読み込まれる可能性があります。退避ファイルは別ディレクトリに置くか、logrotateが読み込まない名前・場所で管理します。

作業後確認

logrotateのdry runで、構文と対象ファイルを確認します。

root shell
logrotate -d /etc/logrotate.conf

タブ補完やシェル設定の反映に問題がある場合は、再ログインまたは再起動後に確認します。
