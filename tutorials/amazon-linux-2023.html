[構築] Amazon Linux 2023

AWS EC2上のAmazon Linux 2023を手動構築する際の初期設定、RHEL9との差分、rsyslog、cron、sudoers、profile、SSH、cloud-initの確認観点を整理します。root権限で実行する前提のため、コマンド例では sudo を省略しています。

前提 実環境名、IPアドレス、顧客固有値は扱わず、再利用できる手順として一般化しています。<HOSTNAME>、<FQDN>、<PRIVATE_IP> などは環境に合わせて置き換えます。

1. RedHat9との差分

項目 Amazon Linux 2023での扱い 構築時の判断
パッケージ管理 dnf が標準です。yum も利用できますが、DNFへのポインタとして扱われます。 手順では dnf に統一します。
SELinux 初期状態は Permissive です。 RHEL9のように permissive へ変更する手順は不要です。状態確認のみ行います。
rsyslog デフォルトではインストールされず、標準ログ確認は journalctl です。 /var/log/messages など従来運用のログファイルが必要な場合は追加導入します。
cron cronie はデフォルトでは含まれません。AWSはsystemd timerへの移行も推奨しています。 既存運用でcrontabを使う場合は cronie を追加導入します。
SSH 古いSSHクライアントでは接続できない場合があります。 クライアント側を更新し、安易に暗号ポリシーをLEGACYへ下げない方針にします。
リポジトリ更新 AMIごとにリポジトリバージョンが紐づき、決定的な更新が重視されています。 本番ではAMI、リポジトリバージョン、更新タイミングを作業前に確認します。

2. 起動時の確認

AWSコンソールでEC2を起動する場合、AMI、インスタンスタイプ、キーペア、Security Group、EBS、IAM Roleを確認します。

確認項目内容
ログインユーザー標準ユーザーは ec2-user です。
キーペアED25519 を基本にします。利用するSSHクライアントが対応しているか確認します。
接続方式SSH、Session Manager、踏み台経由など、作業経路を事前に確認します。
バックアップ設定変更前にAMI、EBS Snapshot、設定ファイル退避の要否を確認します。

3. OSとSELinuxの確認

OSバージョンを確認します。

root shell
cat /etc/os-release

カーネルバージョンを確認します。

root shell
uname -r

SELinuxの状態を確認します。Amazon Linux 2023では初期状態が Permissive のため、RHEL9手順のような設定変更は行いません。

root shell
getenforce
補足 セキュリティ要件で Enforcing が必要な場合は、影響調査、ログ確認、再起動、アプリケーション動作確認をセットで行います。このページでは初期構築の標準手順としては変更しません。

4. パッケージ更新と基本パッケージ導入

パッケージを更新します。

root shell
dnf update -y

運用で使用する基本パッケージを導入します。rsyslog と cronie はAmazon Linux 2023で追加導入が必要になりやすい代表例です。

root shell
dnf install -y vim-enhanced rsyslog cronie cronie-noanacron logrotate \
  tar zip unzip wget telnet traceroute bind-utils sysstat lsof

導入済みパッケージを確認します。

root shell
rpm -q rsyslog cronie cronie-noanacron logrotate sysstat

5. rsyslogの有効化

Amazon Linux 2023は systemd-journald が標準です。従来の /var/log/messages、/var/log/secure、/var/log/cron を使う運用では rsyslog を有効化します。

root shell
systemctl enable --now rsyslog

サービス状態を確認します。

root shell
systemctl status rsyslog

ログファイルの出力状況を確認します。

root shell
ls -l /var/log/messages /var/log/secure /var/log/cron

journald側のログも確認します。

root shell
journalctl -u rsyslog --no-pager

6. cronの有効化

Amazon Linux 2023ではcronがデフォルトでは利用できません。既存運用で crontab を使用する場合は cronie を導入し、crond を有効化します。

root shell
systemctl enable --now crond

サービス状態を確認します。

root shell
systemctl status crond

rootのcrontabを確認します。

root shell
crontab -l

7. sudoers編集時のエディタ指定

visudo や sudo vi /etc/sudoers の実行時にnanoが開く環境では、作業前にエディタを明示します。sudoersは構文ミスの影響が大きいため、直接編集ではなく visudo を使用します。

root shell
export EDITOR=vi
export VISUAL=vi

sudoersを編集します。

root shell
visudo

構文を確認します。

root shell
visudo -c

8. ログインプロファイル設定

操作履歴に時刻を残し、調査時に作業順序を追いやすくするため、/etc/profile と新規ユーザー用の雛形を変更します。

/etc/profile を退避します。

root shell
cp -pi /etc/profile /etc/profile.org

/etc/profile を編集します。

root shell
vi /etc/profile

末尾に履歴時刻とumaskを設定します。

conf
umask 002
HISTSIZE=50000
HISTFILESIZE=50000
HISTTIMEFORMAT='%Y-%m-%d %T '
export HISTSIZE HISTFILESIZE HISTTIMEFORMAT

差分を確認します。

root shell
diff -u /etc/profile.org /etc/profile

9. ホスト名・hosts・時刻設定

必要に応じてホスト名を変更します。

root shell
hostnamectl set-hostname <HOSTNAME>

/etc/hosts を編集します。

root shell
vi /etc/hosts

プライベートIP、FQDN、ホスト名を追記します。

conf
<PRIVATE_IP> <FQDN> <HOSTNAME>

タイムゾーンを設定します。

root shell
timedatectl set-timezone Asia/Tokyo

時刻同期状態を確認します。

root shell
chronyc sources -v

10. SSHとcloud-init確認

SSHサービス状態を確認します。

root shell
systemctl status sshd

古いクライアントで接続できない場合は、まずクライアント側の更新を検討します。やむを得ず互換性を優先する場合のみ、AWS公式手順に沿って暗号ポリシーを変更します。

root shell
dnf install -y crypto-policies-scripts
update-crypto-policies --set LEGACY
注意 LEGACY への変更はセキュリティレベルを下げます。恒久対応ではなく、接続ツールの更新、Session Manager利用、対応済みSSHクライアント利用を優先します。

cloud-initの状態を確認します。

root shell
cloud-init status --long

11. swap設定

メモリ容量やアプリケーション要件に応じてswapを作成します。不要な場合はこの手順をスキップします。

root shell
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

永続化する場合は /etc/fstab に追記します。

root shell
cp -pi /etc/fstab /etc/fstab.org
vi /etc/fstab
fstab
/swapfile none swap sw 0 0

swap状態を確認します。

root shell
swapon --show
free -h

12. logrotate確認

rsyslogやミドルウェアのログをファイル出力する場合は、logrotate設定を確認します。

root shell
logrotate -d /etc/logrotate.conf

個別設定ファイルを確認します。

root shell
ls -l /etc/logrotate.d/
cat /etc/logrotate.d/syslog
